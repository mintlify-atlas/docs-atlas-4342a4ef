## Overview

The Role model extends Discord roles with additional metadata and automation features. It enables level-based role rewards, purchasable roles, referral roles, and various auto-assignment rules.

## Schema Definition

<Note>
The Role model is defined in `src/models/Role.ts` using Mongoose ODM for MongoDB.
</Note>

### Identification

<ResponseField name="_id" type="string" required>
  Discord role ID. This is the primary identifier and must match the actual Discord role ID.
</ResponseField>

<ResponseField name="id" type="string">
  Alternative role ID field (optional duplicate of `_id`)
</ResponseField>

<ResponseField name="guild" type="string" required>
  Discord guild (server) ID where this role belongs
  
  **Reference:** Links to Guild collection
</ResponseField>

### Display Properties

<ResponseField name="description" type="string">
  Custom description for the role, displayed in role selection menus and embeds
  
  **Constraints:** Maximum 100 characters
</ResponseField>

<ResponseField name="emoji" type="string">
  Emoji associated with the role, used in role selection menus
  
  **Formats:** 
  - Unicode emoji: `🎮`
  - Custom emoji: `<:name:id>` or just the emoji ID
</ResponseField>

### Gamification Integration

<ResponseField name="level" type="number">
  Required level for members to automatically receive this role
  
  **Usage:**
  - When a member reaches this level, the role is automatically assigned
  - Requires guild's `gamification` to be enabled
  - Multiple roles can have different level requirements
  
  **Example:** Set to `10` to award this role when members reach level 10
</ResponseField>

### Economy Integration

<ResponseField name="price" type="number">
  Cost in server currency for members to purchase this role
  
  **Features:**
  - Members can buy the role using `/role buy` command
  - Requires guild's `gambling` or economy to be enabled
  - Price is deducted from member's balance
  
  **Example:** Set to `5000` to make the role cost 5,000 coins
</ResponseField>

### Referral System

<ResponseField name="invite" type="string">
  Discord invite code required to receive this role
  
  **Usage:**
  - Members who join through this specific invite get the role
  - Used for partner roles or special access
  - Must be a valid Discord invite code (not full URL)
</ResponseField>

<ResponseField name="referrals" type="number">
  Number of referrals required for members to receive this role
  
  **Usage:**
  - Tracks how many users a member has referred to the server
  - Role is automatically assigned when threshold is reached
  - Encourages server growth through referral rewards
  
  **Example:** Set to `5` to require 5 successful referrals
</ResponseField>

### Automation Settings

<ResponseField name="autoAssignable" type="boolean" default="false">
  When enabled, this role is automatically assigned to new members when they join
  
  **Usage:**
  - Common for default member roles
  - Applies immediately on join
  - Can have multiple auto-assignable roles
</ResponseField>

<ResponseField name="selfAssignable" type="boolean" default="false">
  When enabled, members can assign/remove this role themselves using commands
  
  **Usage:**
  - Used for opt-in roles (notifications, game roles, etc.)
  - Members use `/role add` and `/role remove` commands
  - Popular for color roles and interest groups
</ResponseField>

<ResponseField name="bots" type="boolean" default="false">
  When enabled, this role is specifically for bot accounts
  
  **Usage:**
  - Automatically assigned to bots when they join
  - Helps distinguish bots from regular members
  - Useful for bot-specific permissions
</ResponseField>

## Role Assignment Logic

### Level-Based Assignment

```typescript
import RoleModel from './models/Role';
import MemberModel from './models/Member';

async function checkLevelRoles(userId: string, guildId: string) {
  const member = await MemberModel.findOne({ user: userId, guild: guildId });
  
  // Find all level roles the member is eligible for
  const eligibleRoles = await RoleModel.find({
    guild: guildId,
    level: { $exists: true, $lte: member.level }
  });
  
  // Assign roles to the Discord member
  for (const role of eligibleRoles) {
    await discordMember.roles.add(role._id);
  }
}
```

### Purchase Validation

```typescript
async function purchaseRole(userId: string, guildId: string, roleId: string) {
  const role = await RoleModel.findById(roleId);
  const member = await MemberModel.findOne({ user: userId, guild: guildId });
  
  if (!role.price) {
    throw new Error('This role is not purchasable');
  }
  
  if (member.balance < role.price) {
    throw new Error(`Insufficient balance. Need ${role.price}, have ${member.balance}`);
  }
  
  // Deduct price and assign role
  member.balance -= role.price;
  await member.save();
  
  await discordMember.roles.add(roleId);
}
```

### Auto-Assignment on Join

```typescript
import RoleModel from './models/Role';

// When a new member joins
async function onMemberJoin(member: GuildMember) {
  // Get all auto-assignable roles
  const autoRoles = await RoleModel.find({
    guild: member.guild.id,
    autoAssignable: true
  });
  
  // Assign all auto roles
  const roleIds = autoRoles.map(role => role._id);
  await member.roles.add(roleIds);
  
  // If member is a bot, also add bot roles
  if (member.user.bot) {
    const botRoles = await RoleModel.find({
      guild: member.guild.id,
      bots: true
    });
    
    const botRoleIds = botRoles.map(role => role._id);
    await member.roles.add(botRoleIds);
  }
}
```

<Info>
Role documents are manually created by server administrators or moderators using Bastion commands. They do not auto-create when Discord roles are created.
</Info>

<Warning>
The role ID (`_id`) must exactly match the Discord role ID. If they don't match, automatic role assignment will fail.
</Warning>
