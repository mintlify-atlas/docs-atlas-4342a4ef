Bastion provides a comprehensive set of slash commands to manage your Discord server, engage your community, and enhance the user experience. All commands use Discord's native slash command interface for easy discovery and use.

## How to Use Commands

All Bastion commands are slash commands. Simply type `/` in any channel where Bastion has access, and Discord will show you available commands. You can also type `/` followed by a command name to see its options and parameters.

<Note>
  Commands are organized by category below. Click on any command card to view detailed documentation including parameters, examples, and usage guides.
</Note>

## Command Types

<Tabs>
  <Tab title="Simple Commands">
    Simple commands perform a single action and may accept parameters. Examples include `/about`, `/calculate`, and `/status`.
    
    These commands are straightforward to use and typically provide immediate results.
  </Tab>
  
  <Tab title="Command Groups">
    Command groups contain multiple related subcommands. For example:
    - `/channel` includes `create`, `delete`, `info`, and `update` subcommands
    - `/music` includes `play`, `pause`, `skip`, `queue`, and more
    - `/config` provides extensive configuration options for all features
    
    When using command groups, you'll select a subcommand after typing the main command.
  </Tab>
  
  <Tab title="Context Menus">
    Some Bastion commands are also available as context menu actions (right-click on messages or users).
    
    Context menu commands include:
    - **Sentiment** - Analyze message sentiment
    - **Translate** - Translate message content
  </Tab>
</Tabs>

## Permission Requirements

<Note>
  Many commands require specific Discord permissions to function. Bastion will inform you if you lack the necessary permissions to execute a command. Server administrators can configure command permissions through Discord's server settings.
</Note>

Commands are categorized by their permission requirements:

- **Everyone** - Commands available to all server members
- **Moderators** - Commands requiring moderation permissions
- **Administrators** - Commands requiring administrator permissions
- **Server Owner** - Commands restricted to the server owner
