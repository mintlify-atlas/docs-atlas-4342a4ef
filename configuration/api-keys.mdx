Bastion integrates with numerous third-party services to provide rich features like AI chat, weather information, game statistics, and more. This guide covers all available API integrations and how to configure them.

<Info>
  All API keys are **optional**. Configure only the services you plan to use. Commands that require missing API keys will gracefully indicate they're unavailable.
</Info>

## AI Services

Bastion supports multiple AI providers for the `chat` command, allowing users to interact with AI assistants.

### OpenAI (ChatGPT)

<ParamField path="openai.apiKey" type="string">
  OpenAI API key for ChatGPT integration.
  
  **Get your key**: [OpenAI Platform - API Keys](https://platform.openai.com/api-keys)
  
  ```yaml settings.yaml
  openai:
    apiKey: "sk-proj-..."
    model: "gpt-4o-mini"
    maxTokens: 100
  ```
</ParamField>

<ParamField path="openai.model" type="string" default="gpt-4o-mini">
  The OpenAI model to use for chat responses.
  
  **Available models**:
  - `gpt-4o` - Most capable model, higher cost
  - `gpt-4o-mini` - Fast and affordable, recommended for most uses
  - `gpt-4-turbo` - Previous generation flagship model
  - `gpt-3.5-turbo` - Fast and economical
  
  See [OpenAI Models](https://platform.openai.com/docs/models) for the complete list.
  
  <Info>
    **Pricing**: Different models have different costs per token. Check [OpenAI Pricing](https://openai.com/pricing) for current rates.
  </Info>
</ParamField>

<ParamField path="openai.maxTokens" type="number" default="100">
  Maximum number of tokens in ChatGPT responses.
  
  Controls the length of responses. Higher values allow longer responses but cost more per request.
  
  - `50-100`: Short, concise responses
  - `100-200`: Medium-length responses
  - `200-500`: Detailed responses
  - `500-1000`: Very detailed responses
  
  <Info>
    Use the [OpenAI Tokenizer](https://platform.openai.com/tokenizer) to estimate token counts for your use case.
  </Info>
</ParamField>

**Example configuration**:

```yaml settings.yaml
openai:
  apiKey: "sk-proj-abc123..."
  model: "gpt-4o-mini"
  maxTokens: 150
```

### Google Gemini

<ParamField path="gemini.apiKey" type="string">
  Google AI API key for Gemini integration.
  
  **Get your key**: [Google AI Studio](https://aistudio.google.com/app/apikey)
  
  ```yaml settings.yaml
  gemini:
    apiKey: "AIza..."
    model: "gemini-2.0-flash-exp"
    maxOutputTokens: 256
  ```
</ParamField>

<ParamField path="gemini.model" type="string" default="gemini-2.0-flash-exp">
  The Gemini model to use for chat responses.
  
  **Available models**:
  - `gemini-2.0-flash-exp` - Latest experimental flash model
  - `gemini-1.5-pro` - Most capable model for complex tasks
  - `gemini-1.5-flash` - Fast and efficient for most tasks
  - `gemini-1.0-pro` - Stable production model
  
  See [Gemini Models](https://ai.google.dev/gemini-api/docs/models/gemini) for details.
  
  <Info>
    **Pricing**: Gemini offers generous free tier quotas. Check [Gemini Pricing](https://ai.google.dev/pricing) for limits and costs.
  </Info>
</ParamField>

<ParamField path="gemini.maxOutputTokens" type="number" default="256">
  Maximum number of output tokens in Gemini responses.
  
  Similar to OpenAI's maxTokens, this controls response length.
  
  See [Gemini Tokens Documentation](https://ai.google.dev/gemini-api/docs/tokens) for more information.
</ParamField>

**Example configuration**:

```yaml settings.yaml
gemini:
  apiKey: "AIzaSyA..."
  model: "gemini-2.0-flash-exp"
  maxOutputTokens: 200
```

<ParamField path="anthropic.apiKey" type="string">

<ParamField path="anthropic.model" type="string" default="claude-3-5-haiku-20241022">
  The Claude model to use for chat responses.
  
  **Available models**:
  - `claude-opus-4-20250514` - Most capable model for complex tasks
  - `claude-3-5-sonnet-20241022` - Balanced intelligence and speed
  - `claude-3-5-haiku-20241022` - Fast and affordable, recommended
  - `claude-3-haiku-20240307` - Previous generation fast model
  
  See [Claude Models](https://docs.anthropic.com/en/docs/about-claude/models) for the complete list.
  
  <Info>
    **Pricing**: Claude offers competitive pricing across models. Check [Anthropic Pricing](https://www.anthropic.com/pricing#anthropic-api) for current rates.

<ParamField path="anthropic.maxTokens" type="number" default="128">

## Data & Information Services

### NASA API

<ParamField path="nasaApiKey" type="string" default="DEMO_KEY">
  NASA API key for the `apod` command (Astronomy Picture of the Day).
  
  **Get your key**: [NASA API Portal](https://api.nasa.gov/)

  
  <Info>
    The default `DEMO_KEY` works for testing but has strict rate limits (30 requests per hour, 50 per day). Register for a free API key for production use.
  </Info>
</ParamField>

## Gaming Services

### Tracker Network

<ParamField path="trackerNetworkApiKey" type="string">
  Tracker Network API key for gaming statistics commands.
  
  **Get your key**: [Tracker Network Developers](https://tracker.gg/developers)
  
  ```yaml settings.yaml
  trackerNetworkApiKey: "your-tracker-api-key"
  ```
  
  **Supported commands**:
  - `apex` - Apex Legends player statistics
  - `csgo` - Counter-Strike: Global Offensive stats
  - `fortnite` - Fortnite player statistics
</ParamField>

### Twitch

<ParamField path="twitch" type="object">
  Twitch API credentials for live stream notifications and the `game` command.
  
  **Get your credentials**: [Twitch Developer Console](https://dev.twitch.tv/console/apps)
  
  ```yaml settings.yaml
  twitch:
    clientId: "your-client-id"
    clientSecret: "your-client-secret"
    accessToken: "your-access-token"
  ```
  
  **Setup steps**:
  
  1. Create a Twitch application in the Developer Console
  2. Copy the Client ID
  3. Generate a Client Secret
  4. Generate an OAuth access token using your Client ID and Secret
  
  <Info>
    You can generate an access token using the [Twitch CLI](https://dev.twitch.tv/docs/cli) or the [Twitch Token Generator](https://twitchtokengenerator.com/).
  </Info>
</ParamField>

### Ubisoft (Rainbow Six)

<ParamField path="ubisoft" type="object">
  Ubisoft account credentials for the `rainbow6` command.
  
  ```yaml settings.yaml
  ubisoft:
    email: "your@email.com"
    password: "your-password"
  ```
  
  <Warning>
    **Security Notice**: Storing passwords in configuration files is a security risk. Consider using environment variables or a secrets manager for production deployments.
    
    We recommend creating a dedicated Ubisoft account for bot use rather than using your personal account.
  </Warning>
  
  Provides Rainbow Six Siege player statistics and rankings.
</ParamField>

## Cost Optimization

<Info>
  Most API services offer free tiers sufficient for small to medium Discord bots. Monitor your usage to avoid unexpected costs.
</Info>

### AI Services Cost Tips

1. **Choose the right model**:
   - Use cheaper models (gpt-4o-mini, gemini-flash, claude-haiku) for most interactions
   - Reserve expensive models for complex tasks only

2. **Set appropriate token limits**:
   - Lower `maxTokens` values reduce costs per request
   - 100-200 tokens is usually sufficient for Discord chat

3. **Implement rate limiting**:
   - Use Discord's built-in command cooldowns
   - Set per-user or per-server limits

4. **Monitor usage**:
   - Check API dashboards regularly
   - Set up billing alerts
   - Track usage per command

## Security Best Practices

<Warning>
  API keys are sensitive credentials. Follow these security practices to protect them.
</Warning>

### 1. Use Environment Variables

Store API keys in environment variables instead of `settings.yaml`:

```bash .env
OPENAI_API_KEY=sk-proj-...
GEMINI_API_KEY=AIza...
ANTHROPIC_API_KEY=sk-ant-...
```

Then reference them in your settings or code.

### 2. Never Commit API Keys

- Add `settings.yaml` and `.env` to `.gitignore`
- Use `settings.example.yaml` and `.env.example` as templates
- Never share API keys in public repositories or Discord

### 3. Rotate Keys Regularly

- Change API keys periodically (every 3-6 months)
- Immediately rotate if a key is exposed
- Use separate keys for development and production

### 4. Restrict API Key Permissions

- Use API key restrictions where available (IP whitelisting, domain restrictions)
- Create separate keys for different services
- Grant minimum required permissions

### 5. Monitor Usage

- Set up billing alerts in API provider dashboards
- Review usage logs for suspicious activity
- Revoke unused or old API keys

## Troubleshooting

### Invalid API Key Error

**Solutions**:

1. Verify the API key is correct (no extra spaces or characters)
2. Check if the key has been revoked or expired
3. Ensure the key has proper permissions for the requested service
4. Regenerate the key if necessary

### Rate Limit Exceeded

**Solutions**:

1. Implement command cooldowns in Discord
2. Upgrade to a paid tier if using free tier
3. Cache responses to reduce API calls
4. Use rate limiting middleware

### API Service Unavailable

**Solutions**:

1. Check the service status page
2. Verify your network connection
3. Ensure firewall allows outbound HTTPS (port 443)
4. Try again after a few minutes (temporary outage)

### Authentication Failed (Ubisoft)

**Solutions**:

1. Verify email and password are correct
2. Enable 2FA on your Ubisoft account and use an app password
3. Check if account is locked or requires verification
4. Create a dedicated bot account for reliability
