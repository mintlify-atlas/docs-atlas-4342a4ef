Get your Bastion Discord bot up and running quickly with this streamlined guide. You'll go from zero to running Bastion in just a few steps.

## Before you begin

Make sure you have:
- A Discord account
- A Discord server where you have administrative permissions
- Basic familiarity with Discord's interface

<Note>
  For detailed installation instructions, see the [Installation](/installation) guide.
</Note>

## Get started

<Steps>
  <Step title="Create a Discord application">
    Visit the [Discord Developer Portal](https://discord.com/developers/applications) and create a new application:
    
    1. Click **New Application**
    2. Enter "Bastion" as the application name
    3. Accept Discord's Terms of Service
    4. Click **Create**
  </Step>

  <Step title="Create a bot user">
    In your application settings:
    
    1. Navigate to the **Bot** section in the left sidebar
    2. Click **Add Bot** and confirm
    3. Under the bot's username, click **Reset Token**
    4. Copy the token and save it securely (you'll need this later)
    
    <Warning>
      Never share your bot token publicly. Anyone with your token can control your bot.
    </Warning>
  </Step>

  <Step title="Install Bastion">
    Clone the Bastion repository and install dependencies:
    
    ```bash
    git clone https://github.com/TheBastionBot/Bastion.git
    cd Bastion
    npm install
    ```
    
    <Note>
      You'll need Node.js 18+ and MongoDB running. See [Installation](/installation) for prerequisites.
    </Note>
  </Step>

  <Step title="Configure Bastion">
    Create your configuration file:
    
    ```bash
    cp settings.example.yaml settings.yaml
    ```
    
    Edit `settings.yaml` and add your bot credentials:
    
    ```yaml settings.yaml
    # Bot Token from Step 2
    token: "YOUR_BOT_TOKEN_HERE"
    
    # MongoDB connection
    mongoURI: "mongodb://127.0.0.1:27017/bastion"
    ```
  </Step>

  <Step title="Invite Bastion to your server">
    In the Discord Developer Portal:
    
    1. Go to **OAuth2** → **URL Generator**
    2. Select these scopes:
       - `bot`
       - `applications.commands`
    3. Select these bot permissions:
       - Administrator (or specific permissions you want)
    4. Copy the generated URL and open it in your browser
    5. Select your server and authorize the bot
  </Step>

  <Step title="Register slash commands">
    Build and register Bastion's slash commands:
    
    ```bash
    npm run build
    npm run commands
    ```
    
    This publishes all slash commands to Discord. It may take a few minutes for commands to appear in your server.
  </Step>

  <Step title="Start Bastion">
    Launch your bot:
    
    ```bash
    npm start
    ```
    
    You should see:
    ```
    Shard 0 — Launching [ 1 of 1 ]
    ```
    
    Your bot should now appear online in your Discord server!
  </Step>
</Steps>

## Test your bot

Try running your first command in your Discord server:

```
/about
```

Bastion will respond with information about the bot and links to get started.

<Tip>
  Type `/` in any channel to see all available commands.
</Tip>

## Common issues

<AccordionGroup>
  <Accordion title="Bot doesn't appear online">
    - Verify your bot token is correct in `settings.yaml`
    - Check that MongoDB is running: `mongod --version`
    - Look for error messages in the console output
  </Accordion>

  <Accordion title="Slash commands don't appear">
    - Wait a few minutes for Discord to register commands
    - Ensure you ran `npm run commands` successfully
    - Try kicking and re-inviting the bot with the OAuth2 URL
  </Accordion>

  <Accordion title="Database connection failed">
    - Verify MongoDB is running
    - Check your `mongoURI` in `settings.yaml`
    - For Docker: use `host.docker.internal` instead of `127.0.0.1`
  </Accordion>
</AccordionGroup>

## Production deployment

For production use, consider these options:

<Tabs>
  <Tab title="Bash Script">
    Use the included bash script for easy management:
    
    ```bash
    ./bastion.sh start    # Start Bastion
    ./bastion.sh stop     # Stop Bastion
    ./bastion.sh restart  # Restart Bastion
    ./bastion.sh update   # Update to latest version
    ```
  </Tab>
  
  <Tab title="Docker">
    Run with Docker for isolated deployment:
    
    ```bash
    docker build -t bastion .
    docker run -d --name bastion \
      -v $(pwd)/settings.yaml:/app/settings.yaml \
      bastion
    ```
    
    See [Docker deployment](/deployment/docker) for details.
  </Tab>
  
  <Tab title="Windows">
    Use the PowerShell script:
    
    ```powershell
    .\bastion.cmd
    ```
  </Tab>
</Tabs>
